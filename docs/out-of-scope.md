# 収録対象外（reject）リスト

「思考ツールとして収録するか」を判定した結果、**収録対象外（reject）と判断した提案**を記録します。
同じ提案が繰り返し持ち込まれたときの再審査を防ぎ、境界判定の判例集とするためのものです。

- 受け入れ要件・対象外の定義の正本は [`taxonomy.md`「収録の前提」](./taxonomy.md) を参照。
- ここに載せるのは**ルーチンの却下**。受け入れ要件そのものを変える判断は ADR（[`adr/`](../adr/README.md)）に記録する。
- 後日、要件の見直しや内容の具体化で収録できるようになった場合は、この表から該当行を削除し、通常どおり追加する。

## 記録のしかた

reject と判断したら、下表に1行追加する：

| 列 | 内容 |
| --- | --- |
| 日付 | 判断した日（`yyyy-mm-dd`） |
| 項目（提案された呼称） | 持ち込まれた名前。日本語名がなければ原語のまま |
| 提案された区分 | 提案者がどのタイプのつもりだったか（不明なら「—」） |
| 却下理由 | `taxonomy.md`「収録対象外」のどれに当たるかを一言で |
