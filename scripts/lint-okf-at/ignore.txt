# OKF-AT 準拠チェック (scripts/lint-okf-at/index.js) の除外パターン
#
# .gitignore と同様の記法で、OKF-AT の検査対象から外すファイル・ディレクトリを管理する。
# 除外パターンの正典（SoT）はこのファイル。docs/okf-at/README.md（OKF-AT 仕様）は
# 除外の考え方だけを定め、具体的なパターンはここへの参照で一元管理する（二重管理しない）。
# パターンを追加・変更するときは、理由をコメントで添えること。
#
# 記法（.gitignore のサブセット）:
#   - `#` 始まりはコメント、空行は無視
#   - 末尾 `/` はディレクトリ（配下すべてを除外）
#   - 先頭 `/` または `/` を含むパターンはリポジトリルート起点
#   - `/` を含まないパターンはファイル名としてどの階層でも一致
#   - `*` は `/` を跨がないワイルドカード、`**` は `/` を跨ぐ、`?` は任意の 1 文字
#   - 否定 (`!`) は未対応
#
# なお、OKF の予約ファイル (index.md / log.md) はプロファイル仕様そのものなので
# このファイルではなくスクリプト側で扱う。
