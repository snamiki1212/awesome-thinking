# CLAUDE.md

このファイルは、このリポジトリで作業する際の方針を Claude（および各種コーディングエージェント）に伝えるためのものです。

## このリポジトリについて

「思考」に関する知識を網羅的に一覧管理するためのリポジトリです。
メンタルモデル・思考フレームワーク・思考法・思考術などを収集し、それぞれの違いを定義したうえで、項目ごとに Markdown で整理します。

## 言語方針（最重要）

- **このリポジトリのメイン言語は日本語です。**
- **本文・見出し・説明・README・コミットメッセージ・PR 文面などは、すべて日本語で生成してください。**
- 「アプリ ID」にあたる **ファイル名・ディレクトリ名（スラッグ）は英語（kebab-case）** とします。
  - 理由: URL・git・各種ツールでの文字化けや非互換を避けるため。
  - 例: `mental-models/first-principles.md`（本文タイトルは「第一原理思考」）
- スラッグには対応する日本語名を、各 Markdown のフロントマターと本文タイトルで必ず併記してください。

## ディレクトリ構成

タイプ（思考の種類）ごとにディレクトリを分けます。種類の定義そのものが、ディレクトリの境界になります。

```
README.md                  # トップの一覧（各ジャンルの一覧へのリンク）
docs/taxonomy.md           # 4タイプの定義と違いの整理
mental-models/             # メンタルモデル
  README.md                # メンタルモデルの一覧
  <slug>.md                # 各項目の詳細
frameworks/                # 思考フレームワーク
thinking-methods/          # 思考法
thinking-skills/           # 思考術
CLAUDE.md
```

各タイプの定義は `docs/taxonomy.md` を正本とします。新しい項目を追加する前に必ず参照してください。

## 一覧と詳細の関係

- **一覧（インデックス）**: トップ `README.md` から各ジャンルの `README.md` を辿れます。各ジャンルの `README.md` がそのジャンルの一覧表です。
- **詳細**: 各項目は、そのジャンルのディレクトリ配下の個別 Markdown ファイルに詳細を記載します。

## 項目を追加するときの手順

1. `docs/taxonomy.md` を読み、どのタイプ（ディレクトリ）に属するか判断する。
   - 迷う場合は判断理由を本文の「関連・補足」に書き残す。
2. 該当ディレクトリに `<english-slug>.md` を作成する。
3. 下記のフロントマターと節構成のテンプレートに従う。
4. 該当ジャンルの `README.md` の一覧表に1行追加する。
5. 必要なら関連項目同士を相互リンクする。

### 詳細ファイルのテンプレート

```markdown
---
title: 日本語タイトル
slug: english-slug
type: mental-model | framework | thinking-method | thinking-skill
aliases: [別名, English Name]
tags: [意思決定, 問題解決]   # 任意。目的・ドメインの参考用
---

# 日本語タイトル（English Name）

## 一言でいうと
（1〜2文）

## 定義
（何であるか）

## 使いどころ
（どんな場面で有効か）

## 使い方・手順
（具体的な進め方。箇条書きや番号付きで）

## 例
（具体例）

## 注意点・落とし穴
（誤用やよくある勘違い）

## 関連
- [[関連項目]] へのリンク
```

## 執筆スタイル

- 専門用語には初出時に短い説明を添える。
- 出典が明確な概念（提唱者・書籍など）は分かる範囲で触れる。ただし不確かな出典を断定しない。
- 過度に長くせず、1項目は読み切れる分量に保つ。
