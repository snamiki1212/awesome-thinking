# タイプを L1〜L4 の通し番号で階層化しない

## ステータス
Rejected（4タイプを単一の階層に並べる案を却下）

## 背景・課題
4タイプ（メンタルモデル / 思考フレームワーク / 思考法 / 思考術）に対して、
「Layer があるなら L1〜L4 のように通し番号で階層化したい」という要望があった。
一覧上でタイプ間の上下関係・抽象度を一目で示せると分かりやすい、という狙い。

## 決定
**4タイプを 1本の軸（L1〜L4）に並べる階層化は採用しない。**
タイプはこれまでどおり「並列の4分類」として扱い、Layer 番号は付与しない。

## 理由
`docs/taxonomy.md` の定義上、4タイプは **同じ軸の上に並んでいない** ため。

- 各タイプは「問いの軸」が異なる（何が起きているか / どう整理するか / どの方向から考えるか / どう手を動かすか）。
- このうち **思考法 → 思考フレームワーク → 思考術** は「抽象 → 具体」という1本のグラデーションを成す
  （taxonomy にも `思考法 ⊃ ロジックツリー / MECE` という包含関係の記述がある）。
- しかし **メンタルモデルは別軸**。手順を持たない「静的なレンズ（何が起きているかを見る視点）」であり、
  上記の「行為の抽象度」グラデーションには乗らない。

4タイプすべてを L1〜L4 に通し番号付けすると、本来別軸であるメンタルモデルを
無理やり同じ梯子に乗せることになり、タイプ間の関係を誤って表現してしまう。

## 検討した代替案
- **案A: 全4タイプを L1〜L4 で階層化** → 却下（上記理由。別軸を1軸に潰してしまう）。
- **案B: 行為系3タイプのみ「上位/中位/下位」を補足表記** → 将来検討の余地あり。
  思考法（上位の構え）→ フレームワーク（型に落ちたもの）→ 思考術（その場の技）の
  抽象度関係は taxonomy 本文で言及済みのため、必要になれば別 ADR で扱う。
- **案C: 番号を付けず現状維持（並列の4分類）** → 採用。

## 影響
- 一覧（README・各 README）では引き続きタイプを並列に表示し、L1〜L4 表記は導入しない。
- 抽象度の関係は `docs/taxonomy.md` の本文説明に委ねる。
- 行為系3タイプの段階表記が必要になった場合は、本 ADR を参照のうえ新規 ADR を起こす。
