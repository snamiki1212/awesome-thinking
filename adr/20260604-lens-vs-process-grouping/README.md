# 思考ツールを「レンズ系（1）」と「プロセス系（3）」に大別する

## ステータス
Accepted（4タイプの最上位の切れ目を「1 対 3」とし、2つの中間カテゴリ名を採用）

## 背景・課題
4タイプ（メンタルモデル / 思考フレームワーク / 思考法 / 思考術）は並列に提示してきたが、
読者から「4つ並べても違いが掴みにくい」「むしろ 1+3 で捉えた方が分かりやすいのでは」という声があった。

実際、これまでの整理で4タイプには**最も大きな切れ目が1か所**あることが分かっている。

- **思考法 → フレームワーク → 思考術** は「抽象 → 具体」の1本のグラデーション（入れ子の包含関係）。
  いずれも「**どう考え、どう手を動かすか**」という行為に関わる。
- **メンタルモデルだけは別軸**。手順を持たない「静的なレンズ」で、「**何が起きているか**」を捉える視点。
  この行為の抽象度グラデーションには乗らない（→ [20260602-no-layer-numbering](../20260602-no-layer-numbering/README.md)）。

総称「思考ツール」（→ [20260604-umbrella-term-thinking-tools](../20260604-umbrella-term-thinking-tools/README.md)）は決まっているが、
その下で4タイプをどうグルーピングして見せるかが未定だった。

## 決定
思考ツールの最上位の切れ目を **「1 対 3」** とし、間に2つの中間カテゴリを置く。
それぞれの名前を以下に**採用**する。

- **レンズ系** … 🧠 メンタルモデル（1）
  - 状況を**見る・捉える**ための静的な視点。「何が起きているか」を捉える。手順はない。
- **プロセス系** … 🌊 思考法 ⊃ 🧩 フレームワーク ⊃ ⚡ 思考術（3）
  - 実際に**考えを進める・手を動かす**ための道具。「どう考え、どう手を動かすか」を与える。
  - 内部は「抽象（構え）→ 具体（その場の手）」の入れ子。

```
思考ツール（4タイプの総称）
├─ レンズ系   … 🧠 メンタルモデル                         ＝ 見る／捉える（静的）
└─ プロセス系 … 🌊 思考法 ⊃ 🧩 フレームワーク ⊃ ⚡ 思考術   ＝ 進める／動かす（抽象→具体）
```

本決定は **提示・説明上の中間カテゴリ（概念区分）の取り決め**であり、
ディレクトリの新設・階層化は伴わない（フラット構成は
[20260604-flat-directory-structure](../20260604-flat-directory-structure/README.md) のとおり維持）。

## 理由
- 4タイプの違いの根っこは「**静的に"見る"道具か／動的に"考え・動かす"道具か**」の一点であり、
  これを最上位の切れ目に据えると「まず1対3」で直感的に掴める。
- この 1+3 は [20260602-no-layer-numbering](../20260602-no-layer-numbering/README.md) の理由
  （メンタルモデルだけは別軸で、行為の抽象度グラデーションに乗らない）と完全に整合する。
- 「レンズ／プロセス」は専門知識がなくても意味が伝わる平易な語で、リポジトリのトーンに合う。
- 既存のタイプ名（メンタルモデル等）やディレクトリ名と語が衝突しない。

## 検討した代替案

### グルーピングの方針
- **A: 1+3（レンズ系 / プロセス系）** → 採用。最上位の切れ目が1つで掴みやすい。
- **B: 4タイプを並列のまま（中間カテゴリなし）** → 見送り。現状の「違いが掴みにくい」課題が残る。
- **C: 全4タイプを L1〜L4 の1列に並べる** → 却下済み（別軸のメンタルモデルを梯子に乗せてしまう。
  [20260602-no-layer-numbering](../20260602-no-layer-numbering/README.md)）。

### カテゴリ名（採用した「レンズ系 / プロセス系」以外の候補）
- グループ1（メンタルモデル）側
  - **視点ツール / 認識系** → 見送り。意味は正確だがやや硬い。
  - **静的ツール** → 見送り。対比は明快だが無機質で初学者向きでない。
- グループ3（思考法・フレームワーク・思考術）側
  - **実践系 / 操作系** → 見送り。悪くないが「レンズ」と対にしたとき分かりにくい。
  - **動的ツール** → 見送り。「静的ツール」と対だが同上の理由で硬い。

「レンズ（見る）／プロセス（進める）」という日常語の対が、最も直感的で対比も明快だったため採用。

## 影響
- README・`docs/taxonomy.md` の図や説明で、4タイプを **「レンズ系（1）／プロセス系（3）」** の
  2グループとして提示してよい（既存の入れ子図・比較表と矛盾しない）。
- ディレクトリ構成・スラッグ・タイプ定義は**変更しない**（提示上の中間カテゴリの取り決めのみ）。
- L1〜L4 のような通し番号の階層化は引き続き行わない（本決定は番号梯子ではなく「1 対 3」の区分）。
- 反映作業（図・本文への併記）は本 ADR を受けて別途行う。
