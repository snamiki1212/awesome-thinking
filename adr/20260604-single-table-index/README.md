# トップ README の一覧を単一テーブルに統合する

## ステータス
Accepted（採用）

## 背景・課題
トップ `README.md` の「一覧」は、これまで4タイプ（メンタルモデル / 思考フレームワーク / 思考法 / 思考術）ごとに見出しを分け、それぞれ別テーブルで項目を並べていた。
項目数が増えるとセクションが縦に伸び、ジャンルをまたいだ俯瞰（全項目の一覧性）が得にくいという課題があった。

## 決定
**トップ README の一覧を1つのテーブルに統合する。**

- ジャンルは独立した「ジャンル」列で表し、絵文字付き（🧠 / 🧩 / 🌊 / ⚡）で判別できるようにする。
- ジャンル別の一覧（各ジャンルの `README.md`）へのリンクは、テーブルとは分離してテーブル上部にまとめて配置する。

## 理由
- 全項目を1つの表で俯瞰でき、ジャンル横断の一覧性が上がる。
- ジャンルは列として残るため、分類情報は失われない。絵文字でジャンルを視覚的に素早く判別できる。
- 「全項目の俯瞰（単一テーブル）」と「ジャンル単位の深掘り（各 README へのリンク）」は目的が異なるため、後者をテーブルから分離することで両者の役割が明確になる。

## 検討した代替案
- **案A: ジャンルごとに見出し＋個別テーブル（従来）** → 却下。項目増加で縦長になり、横断的な一覧性に欠ける。
- **案B: ジャンル列を持つ単一テーブル＋一覧リンクは分離** → 採用。
- **案C: ジャンルでセクション分けしつつ、トップに全項目の統合テーブルも併記** → 却下。同じ情報の二重管理になり、追加時の更新コストと不整合リスクが増える。

## 影響
- トップ `README.md` の「一覧」は単一テーブル方式に統一する。
- 項目を追加する際は、トップ README の統合テーブルにジャンル列付きで1行追加する（各ジャンルの `README.md` への追記手順は従来どおり）。
- 各ジャンルの `README.md`（ジャンル別の一覧）は引き続き維持し、トップからはリンクで辿れる。
