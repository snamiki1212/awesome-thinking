# 思考の4タイプをトップ直下にフラットに置く（thinking/ でまとめない）

## ステータス
Accepted（フラット構成を維持。`thinking/` でのグルーピングは見送り）

## 背景・課題
現状、思考の4タイプ（`mental-models/` / `frameworks/` / `thinking-methods/` / `thinking-skills/`）は
リポジトリのトップ直下に並列で置かれている。

これに対し、「コンテンツを `thinking/` のような親ディレクトリでまとめてはどうか」という案が出た。
例: `thinking/frameworks/` `thinking/mental-models/` …。
トップ直下が「コンテンツ」と「メタ（`adr/` `docs/` `README.md`）」で分離され、
4タイプが “思考” の下位概念であることを階層でも表現できる、という狙い。

## 決定
**4タイプはこれまでどおりトップ直下にフラットに置く。**
`thinking/` などの親ディレクトリでまとめる階層化は、現時点では採用しない。

## 理由
- **リポジトリ自体（`awesome-thinking`）が丸ごと「思考」をテーマにしている**ため、
  `thinking/` で包むのは同義反復になり、その階層が新たな情報を持たない。
- 本質的に意味を持つ軸は **「4タイプの違い」**（`docs/taxonomy.md` が正本とする境界）であり、
  それはトップ直下に4つ並べた方がストレートに伝わる。
- パスが1段深くなる分、Markdown 間のリンクやファイル参照のコストが増える。
- 「メタとコンテンツを分けたい」という動機自体は妥当だが、それはコンテンツを包むより
  **メタ側を独立させる**ことで満たせる（`adr/` `docs/` は既にメタとして分離済み）。
- 項目数がまだ少なく、フラットな見通しの良さの利得が大きい。

## 検討した代替案
- **案A: `thinking/<type>/` でコンテンツをまとめる** → 却下（上記理由。同義反復で利得が薄い）。
- **案B: フラット維持（トップ直下に4タイプ）** → 採用。

## 影響
- ディレクトリ構成は現状維持。`CLAUDE.md` の「ディレクトリ構成」も変更しない。
- 将来、**思考以外のドメイン**（例: 行動・習慣など別ジャンル）を同居させる構想が出てきた場合は、
  その時点で `thinking/` によるグルーピングを価値あるものとして再検討する。その際は本 ADR を参照のうえ新規 ADR を起こす。
