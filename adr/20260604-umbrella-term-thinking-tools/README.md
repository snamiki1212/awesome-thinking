# 4タイプを束ねる総称を「思考ツール（thinking tools）」とする

## ステータス
Accepted（中間概念の呼称として「思考ツール / thinking tools」を採用）

## 背景・課題
このリポジトリは思考に関する知識を **4タイプ**
（メンタルモデル / 思考フレームワーク / 思考法 / 思考術）に分けて整理している。
これら4つを **一段抽象化してまとめて指す「総称（親概念）」をどう呼ぶか** が未定だった。

当初は「`thinking`（思考）でよいのでは」という案が出たが、`thinking` は
**リポジトリ全体の総称**としてすでに使われており、また「思考」という語は
行為・状態を指すため、「考えるための型・道具の集合」を指す中間概念の名前としては
ややズレる（同義反復にもなりやすい）。

## 決定
4タイプを束ねる中間概念の総称を **「思考ツール（thinking tools）」** とする。
リポジトリ全体の呼称としての `thinking` / 「思考」はこれまでどおり温存し、
**「リポジトリ全体の総称＝thinking」「4タイプを束ねる中間概念＝思考ツール」** と役割を分ける。

なお本決定は **呼称（概念名）の取り決め**であり、ディレクトリの新設・階層化は伴わない
（フラット構成は [20260604-flat-directory-structure](../20260604-flat-directory-structure/README.md) のとおり維持）。

## 理由
- 4タイプはいずれも「考えるための道具・型」であり、`docs/taxonomy.md` 自身が
  これらを繰り返し「理解の道具」「考えるための型・道具」と説明している。
  共通項を最も的確に表すのが「**道具（tools）**」である。
- 既存タイプ名との衝突を避けられる。
  - 「思考の枠組み（thinking frameworks）」は `frameworks/`（思考フレームワーク）と衝突。
  - 「思考技法（thinking techniques）」は `thinking-skills/`（思考術）と衝突。
- リポジトリ総称 `thinking` と中間概念を別語にすることで、両者の役割が分離され明快になる。
- 平易な日本語＋英語 kebab スラッグというリポジトリの方針・トーンに合う。

## 検討した代替案
- **thinking / 思考** → 却下。リポジトリ総称と重複し同義反復。行為を指す語で集合の名に不向き。
- **thinking-toolkit / 思考の道具箱** → 見送り。「入れ物（箱）」のニュアンスが強く、概念名より UI 寄り。
- **cognitive-tools / 認知ツール** → 見送り。正確だが硬く、平易なトーンと不一致。
- **mind-tools / 知の道具** → 見送り。雰囲気はよいが意味が広く曖昧。
- **thinking-patterns / 思考パターン** → 見送り。メンタルモデルや「構え」をパターンと呼びにくい。
- **thinking-frameworks / 思考の枠組み** → 却下。タイプ名 `framework` と衝突。
- **thinking-techniques / 思考技法** → 却下。タイプ名「思考術」と衝突。
- **thinking-types / 思考のタイプ** → 却下。「分類の単位」は指せても「束ねた概念」を指せない。

## 影響
- 文章中で4タイプをまとめて指すときの語を「思考ツール（thinking tools）」に統一できる。
  README・taxonomy 等で総称が必要になった箇所はこの語を用いる。
- ディレクトリ構成・スラッグは変更しない（呼称のみの取り決め）。
- 将来、思考以外のドメインを同居させる場合は、この総称の射程を再検討し、必要なら新規 ADR を起こす。
