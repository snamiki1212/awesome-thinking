# 英語名など多言語の正式名を frontmatter の titles で管理する

## ステータス

Accepted（採用）

## 背景・課題

各思考ツールの英語名は、これまで frontmatter の `aliases` と本文 H1 の括弧書き（`# 日本語名（English Name）`）に散在し、次の問題があった。

- `aliases` の中で「英語の正式名」と「単なる別名・略称」が区別できず、機械的に英語名を取り出せない
- H1 の括弧書きも英語名だったり日本語の別名だったり（例: `クリティカルシンキング（批判的思考）`）で一貫しない
- トップ README の一覧に英語名がなく、英語名で項目を探せない

また、英語以外の言語（例: 思考実験のドイツ語名 Gedankenexperiment）の正式名を置ける拡張性も欲しい。

## 決定

1. frontmatter に拡張フィールド **`titles`** を設ける。言語コード（ISO 639-1、小文字）をキーに、その言語での正式名を書くマップとする。英語名は `titles.en`。任意フィールドだが、思考ツールの詳細ファイルでは付ける。
2. `title`（日本語の正式名・string）は従来どおり必須。`titles` に `ja` は書かない（`title` が正で、二重管理を避けるため）。
3. `aliases` は正式名以外の「別名・略称・別表記」専用とし、`title` / `titles` に載せた正式名は重複させない。
4. 本文 H1 の括弧書きは `titles.en` と揃え、`# 日本語名（English Name）` とする。
5. トップ README の一覧の項目欄は `[日本語名](パス)（English Name）` と英語名を併記する。日本語タイトルと英語名が同一表記の項目（MECE・KPT・5W1H など）は括弧を省略する。

## 理由

- `titles` に正式名、`aliases` に別名と役割を分けることで、エージェント・ツールが言語ごとの正式名を機械的に取り出せる
- 言語コードをキーにしたマップなので、`en` 以外の言語も規約を変えずに追加できる
- OKF v0.1 は `title` を string と定めている。`titles` を別の拡張フィールドにすれば、`title` の型を変えずに済み、「OKF-AT 準拠なら OKF v0.1 にも準拠する」という不変条件を保てる

## 検討した代替案

- **`title` をマップにする（`title: {ja: ..., en: ...}`）**: OKF v0.1 の `title: string` と非互換になり、`title` を読む既存ツール（gen-okf-index・lint）もすべて壊れるため却下。
- **`title_en` のようなフラットなフィールド**: 言語が増えるたびにフィールドが増え、規約の改定が必要になるため却下。
- **`aliases` のままにする（現状維持）**: 正式名と別名が区別できない課題が解決しないため却下。
- **`titles` に `ja` も含める**: `title` との二重管理になり、片方の更新漏れで食い違うため却下。

## 影響

- `docs/okf-at/README.md` に `titles` の規約（フィールド表・使い分けの節）を追加した。lint（`scripts/lint-okf-at`）は必須フィールドのみ検査する方針のまま変えない（`titles` は任意のため）。
- 既存の全思考ツール詳細ファイル（35件）に `titles.en` を追加し、`aliases` から英語の正式名を取り除いた。H1 の括弧書きも英語名に統一した。
- トップ README の一覧と、スキル `add-thinking-concept` のテンプレート・手順を新しい表記に更新した。今後の項目追加では `titles.en` の記入と README への英語名併記が手順に含まれる。
