# 思考ツールの階層と呼称を整理する（総称＝思考ツール／系統／タイプ）

## ステータス
Accepted（採用）

## 背景・課題
これまでの一連の決定で、思考ツールには **3つの階層** が生まれた。

- 全体の総称（→ [20260604-umbrella-term-thinking-tools](../20260604-umbrella-term-thinking-tools/README.md)）
- 「1＋3」のグループ（→ [20260604-lens-vs-process-grouping](../20260604-lens-vs-process-grouping/README.md)）
- 個々の4つ

ところが中間のグループを **「区分」「中間カテゴリ」「2系統」** とバラバラに呼んでおり、
個々の4つを「タイプ」と呼ぶことも明文化されていなかった。
階層ごとの**呼称を1つに確定**し、README・図・ADR で統一したい。

## 決定
思考ツールの階層と呼称を、次の3段で確定する。

| 階層 | 呼称 | 数 | 中身 |
| --- | --- | --- | --- |
| 総称 | **思考ツール** | 1 | 4つすべてを束ねる総称 |
| 中間 | **系統** | 2 | 🔍 レンズ系 ／ ⚙️ プロセス系 |
| 最小 | **タイプ** | 4 | 🧠 メンタルモデル ／ 🧩 思考フレームワーク ／ 🌊 思考法 ／ ⚡ 思考術 |

- 数えるときは「**2系統**」「**4タイプ**」と言う。
- 中間グループを指す語は **「系統」に統一**し、今後「区分」「中間カテゴリ」等は使わない。
- 包含関係は **思考ツール ⊃ 系統 ⊃ タイプ**。
  （プロセス系の内部はさらに 🌊 思考法 ⊃ 🧩 フレームワーク ⊃ ⚡ 思考術 の抽象→具体の入れ子。）

正本は [`CLAUDE.md` 用語節](../../CLAUDE.md) と [`docs/taxonomy.md`](../../docs/taxonomy.md)。

## 理由
- **「系統」**: 2つのグループはそれぞれ「レンズ**系**」「プロセス**系**」と命名済み。
  その集合を指す語としては「系統」が最も自然で、メンバー名と一貫する。
  「区分」「カテゴリ」は意味が一般的すぎて、どの階層の区分かが伝わりにくい。
- **「タイプ」**: 「4タイプ」は README 見出し・本文・スキルで既に定着しており、変更コストが高く利得がない。
- 3段が「総称／系統／タイプ」と語尾も粒度も揃い、口頭でも数え方（2系統・4タイプ）で階層を即指定できる。

## 検討した代替案

### 中間グループの呼称（採用＝「系統」）
| 候補 | 評価 |
| --- | --- |
| 系統 | **採用**。メンバー名「◯◯系」と一貫し、集合を自然に指せる |
| 区分 | 却下。一般的すぎて階層が曖昧。これまで揺れていた語の一つ |
| 中間カテゴリ | 却下。説明的で長く、固有の呼称に向かない |
| 大分類 / グループ | 見送り。意味は通るが「系」の命名と響き合わない |

### 個々の4つの呼称（採用＝「タイプ」）
| 候補 | 評価 |
| --- | --- |
| タイプ | **採用**。既に定着。「4タイプ」で通っている |
| 種類 / 分類 | 見送り。意味は同じだが既存表記を覆す利得がない |

## 影響
- README・`docs/taxonomy.md`・図・新規ADR では中間グループを **「系統」** に統一する
  （例: 「2系統」「レンズ系／プロセス系」「系統の列」）。
- 4タイプ比較表に **「系統」列** を追加してよい。
- 既にマージ済みの ADR [lens-vs-process-grouping](../20260604-lens-vs-process-grouping/README.md) の
  「中間カテゴリ」表記は**記録としてそのまま残す**が、読み替えは本 ADR を正本とする。
- ディレクトリ構成・タイプ定義は変更しない（呼称の取り決めのみ）。
