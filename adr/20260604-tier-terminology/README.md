# 思考ツールの階層の呼称をカタカナ英語で揃える（総称＝思考ツール／グループ／タイプ）

## ステータス
Accepted（採用）

## 背景・課題
思考ツールは **3階層**（総称 ／ 中間グループ ／ 個々の4つ）で構成される。これまで呼称が定まらず、次の問題があった。

1. 中間グループを「区分」「中間カテゴリ」「系統」と**バラバラ**に呼んでいた。
2. 最小階層を「**タイプ**」（カタカナ英語）と呼ぶ一方、中間を「**系統**」（漢語）と呼んでおり、**語種が混在して平仄が合わない**。
3. メンバー名「レンズ**系** ／ プロセス**系**」の「**系**」は、まさに置き換えたい『系**統**』の字が残り、座りが悪い。

→ 階層ラベルもメンバー名も **カタカナ英語へ寄せて統一**する。

## 決定
階層と呼称を次の3段で確定する。

| 階層 | 呼称 | 数 | メンバー |
| --- | --- | --- | --- |
| 総称 | **思考ツール** | 1 | 4つすべてを束ねる総称 |
| 中間 | **グループ** | 2 | 🔍 レンズ ／ ⚙️ プロセス |
| 最小 | **タイプ** | 4 | 🧠 メンタルモデル ／ 🧩 思考フレームワーク ／ 🌊 思考法 ／ ⚡ 思考術 |

- 中間階層の呼称は **「グループ」**。数えるときは「**2グループ／4タイプ**」。
- メンバー名から **「系」を外す**：レンズ系 → **レンズ**、プロセス系 → **プロセス**。
  文脈上グループだと明示したいときは「レンズグループ」「プロセスグループ」と書いてよい。
- 包含関係は **思考ツール ⊃ グループ ⊃ タイプ**。
  （プロセスグループの内部はさらに 🌊 思考法 ⊃ 🧩 フレームワーク ⊃ ⚡ 思考術 の抽象→具体の入れ子。）
- 「区分」「中間カテゴリ」「系統」「〜系」は今後使わない。

正本は [`CLAUDE.md` 用語節](../../CLAUDE.md) と [`docs/taxonomy.md`](../../docs/taxonomy.md)。

## 理由
- **平仄**: 「グループ」「タイプ」はどちらもカタカナ英語で語種が揃う。漢語の「系統」を避けられる。
- **包含の自明さ**: 「グループ ⊃ タイプ」は『型をまとめた集まり』という上下関係が言葉から自明。
  「カテゴリ」「クラス」は「タイプ」と意味が近く、どちらが上位か曖昧になりやすい。
- **「系」を外す**: 置き換え対象の漢字（系統の「系」）がメンバー名に残らず、最もスッキリする。
- **「タイプ」は維持**: 「4タイプ」は README 見出し・本文・スキルで既に定着しており、覆す利得がない。

## 検討した代替案

### 中間グループの呼称（採用＝「グループ」）
| 候補 | 評価 |
| --- | --- |
| グループ | **採用**。「型の集まり」という包含が明快で、タイプと意味が被らない。カタカナで平仄も合う |
| カテゴリ | 見送り。一般的だが「タイプ」と意味が近く、上下関係が曖昧 |
| クラス | 見送り。同上に加え、プログラミング用語との混同 |
| ファミリー | 見送り。生物分類風で正確だが、ビジネス文ではやや硬い・大げさ |
| 系統（漢語） | 却下。「タイプ」（カタカナ）と平仄が合わない（本件の主因） |
| 区分 / 中間カテゴリ | 却下。一般的すぎ／説明的で、固有の呼称に向かない |

### メンバー名の「系」の扱い（採用＝「外す」）
| 案 | 評価 |
| --- | --- |
| 「系」を外す（レンズ ／ プロセス） | **採用**。置換対象の漢字が残らず最もスッキリ |
| 「系」を残す（レンズ系 ／ プロセス系） | 却下。『系統』の字が残り座りが悪い（今回の動機に反する） |
| グループ語を付ける（レンズグループ 等） | 見送り。正確だが列・図では長い。必要時の補助表記に留める |

### 個々の4つの呼称（採用＝「タイプ」）
| 候補 | 評価 |
| --- | --- |
| タイプ | **採用**。既に定着。「4タイプ」で通用 |
| 種類 / 分類 | 見送り。意味は同じだが既存表記を覆す利得がない |

## 影響
- README・`docs/taxonomy.md`・図・関連ADR の表記を **「グループ／タイプ」「🔍 レンズ ／ ⚙️ プロセス」** に統一する。
- 4タイプ比較表の「系統」列は **「グループ」列** に改名する。
- マージ済み ADR [lens-vs-process-grouping](../20260604-lens-vs-process-grouping/README.md) の
  「レンズ系／プロセス系」「中間カテゴリ」表記は**記録としてそのまま残す**が、読み替えは本 ADR を正本とする。
- ディレクトリ構成・タイプ定義は変更しない（呼称の取り決めのみ）。
