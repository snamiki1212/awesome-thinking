# 項目名は特定出典の造語より一般的・確立した呼称を優先する

## ステータス
Accepted（採用）

## 背景・課題
「不確実性」に関する思考法を追加する際、当初は広木大地『エンジニアリング組織論への招待』の整理だけに依拠して **「不確実性の削減（Uncertainty Reduction）」** という名前で起案した。

レビューの過程で次の問題が明らかになった。

- **単一出典の造語を一般語のように提示していた**: 「不確実性を削減する」という発想自体は、不確実性コーン・アジャイルの経験主義・リスク管理・情報理論など**国際的に確立**しているのに、本文は一冊の本のフレーミング（「環境不確実性／通信不確実性」の2分類を含む）だけを唯一の出典のように書いていた。
- **より一般的な呼称があった**: 国際的な実務・文献で通用するのは「削減（reduction）」より **「マネジメント／管理（managing uncertainty）」** という上位の語だった。
- **英語名が別分野の確立概念と衝突していた**: `Uncertainty Reduction` は対人コミュニケーション論の確立理論「Uncertainty Reduction Theory（Berger & Calabrese, 1975）」を指す名前でもあり、取り違えを招く。
- **内部の収まりの良さを命名根拠にしていた**: 議論の途中、「削減」を残す理由として *既存の[ネガティブ・ケイパビリティ](../../thinking-methods/negative-capability.md)（抱える）との対比が綺麗だから* を挙げていた。これは概念そのものの一般性・正確さではなく、**リポジトリ内の収まり・対比の良さ**という内部都合であり、命名・採否の根拠としては不適切だった。

既存の方針「不確かな出典を断定しない」（`CLAUDE.md`）はあったが、**「確立はしているが特定著者の造語に寄った呼称」を一般語のように採ってしまう**ケースや、**既存項目との対比の良さで命名・採否を決めてしまう**ケースを防ぐ指針が明文化されていなかった。

## 決定
項目の命名・出典の扱いについて、次を方針として定める（手順の正本はスキル `add-thinking-concept`）。

1. **一般的・確立した呼称を優先する**: 日本語タイトル・英語名は、特定の著者・一冊の本の造語より、その分野で国際的・一般的に通用する呼称を選ぶ。
2. **出典は単一に依存しない**: 可能なら複数・国際的な裏づけを併記する。特定著者の整理は「整理の一つ」として明示し、汎用の確立概念であるかのように断定しない（「不確かな出典を断定しない」の具体化）。
3. **英語名の衝突を確認する**: `aliases` を含む英語名は、別分野で確立した同名・類似名の概念と紛れないかを確認し、紛らわしければ避ける。
4. **内部の収まりで命名・採否を決めない**: 項目名・タイプ・採否は、その概念自体の一般性・正確さ・出典で決める。「既存項目との対比が綺麗」「リポジトリに収まりがよい」といった**内部都合を根拠にしない**。既存項目との対比・補完は `関連`／`似ている用語との違い` 節で表現すれば足りる。

実例として、上記の項目は **「不確実性マネジメント（Managing Uncertainty）」** に確定し、広木の2分類は「分かりやすい整理の一つ」として位置づけ直した。

## 理由
- **正確さ**: 一冊の本のフレーミングを一般語のように出すと、読者に「確立した定訳」と誤認させる。複数出典に当たることで、何が一般で何が特定著者の整理かを切り分けられる。
- **発見可能性・取り違え防止**: 一般的な呼称・衝突しない英語名にしておくと、別分野の同名概念との混同を防げる。
- **判断軸を外部に置く**: 命名・採否を「リポジトリ内の対比の良さ」で決めると、概念の実体より自分たちの都合が優先され、名前や分類が歪む。判断軸は常に概念そのもの（一般性・正確さ・出典）に置く。
- **既存方針との整合**: 「不確かな出典を断定しない」（`CLAUDE.md` 執筆スタイル）の自然な延長であり、手順の正本をスキルに置く方針（[procedure-in-skill](../20260604-procedure-in-skill/README.md)）とも整合する。

## 検討した代替案
- **「不確実性の削減」のまま残す**: 既存の[ネガティブ・ケイパビリティ](../../thinking-methods/negative-capability.md)（抱える）との対比が綺麗、という理由が挙がったが、それは内部の収まりであって命名根拠にはならない。より一般的な呼称があるのに造語寄りの名前を採るのは本方針に反するため見送り。「削減」は「マネジメント」の中の一手として本文に残した（対比は `関連`／`似ている用語との違い` で表現すれば足りる）。
- **reject する**: テーマ自体は国際的に確立しているため収録対象であり、reject は不適。
- **ルール化せず個別対応する**: 同種の取り違え（単一出典の造語・英語名衝突）は再発しやすく、判断がぶれるため明文化を選んだ。

## 影響
- スキル `add-thinking-concept` の手順2（スラッグ・命名）に、命名・出典・英語名衝突のチェックを追記。
- スキルの執筆スタイル要点と完了チェックリストに、出典の複数確認と英語名衝突確認の項目を追加。
- `CLAUDE.md` 執筆スタイルに、一般的呼称の優先と単一出典に依存しない旨を1行追記。
- 今後、出典が特定の著者・書籍に偏る項目を追加する際は、まず一般的な呼称・国際的な裏づけ・英語名の衝突を確認してから命名する運用とする。
