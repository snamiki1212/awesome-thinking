# 項目追加の手順・テンプレートはスキルを正本とする

## ステータス
Accepted（採用）

## 背景・課題
新しい思考概念を追加する手順を、コーディングエージェント向けにスキル
`add-thinking-concept`（`.claude/skills/add-thinking-concept/SKILL.md`）として切り出した。
しかし切り出し後も、同じ「項目追加の手順」と「詳細ファイルのテンプレート」が `CLAUDE.md` にも残っており、
**同一知識が2か所に重複**していた。片方だけ更新すると内容がズレ、どちらが正しいか分からなくなる。

## 決定
項目追加の **手順とテンプレートの正本はスキル `add-thinking-concept` の1か所** とする。
`CLAUDE.md` 側からは手順・テンプレートの本体を削除し、スキルへの短いポインタに置き換える。

## 理由
- **単一の正本（DRY）**: 手順・テンプレートを1か所に集約し、二重管理による不整合を防ぐ。
- **役割分担が明確になる**:
  - `CLAUDE.md` は毎セッション必ずコンテキストに載る「常時オンの規約・方針」。短く保つほど良い。
  - スキルは必要なとき（項目追加時）だけ展開される「手順書」。段階的開示の対象として手順・テンプレートが適切。
- タイプ判定の基準は `docs/taxonomy.md`、言語・命名の方針は `CLAUDE.md`「言語方針」を引き続き正本とし、
  スキルはそれらを参照する立場を保つ（正本の所在が概念ごとに1つに定まる）。

## 検討した代替案
- **CLAUDE.md に手順を残し、スキルを薄いラッパ／廃止にする**: 常時コンテキスト保証は得られるが、
  わざわざ作ったスキルの意義が薄れ、段階的開示の利点を捨てることになるため却下。
- **現状維持（両方に手順を持つ）**: 二重管理の不整合リスクが残るため却下。

## 影響
- `CLAUDE.md`「項目を追加するときの手順」は、手順本体とテンプレート節を削除し、スキルへのポインタに変更。
- 今後、項目追加の手順やテンプレートを変えるときは **スキル側のみを更新** すればよい。
- 人が手作業で追加する場合も、スキルの `SKILL.md` を手順書として読む運用とする。
