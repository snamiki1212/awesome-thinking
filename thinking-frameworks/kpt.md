# KPT（Keep / Problem / Try）

## 一言でいうと
続けること・課題・次に試すことの3つで振り返りを整理する枠組み。

## 定義
Keep（良かった・続けること）/ Problem（問題・課題）/ Try（次に試すこと）の3観点で振り返り（レトロスペクティブ）を構造化するフレームワーク。

## 使いどころ
- チームや個人の定期的な振り返り。
- プロジェクトや施策の区切りでの学びの整理。

## 使い方・手順
1. **Keep**: うまくいったこと・今後も続けたいことを挙げる。
2. **Problem**: 困ったこと・課題を挙げる。
3. **Try**: Problem への対策や、新たに試したいことを挙げる。
4. 次回、前回の Try が Keep になったかを確認する。

## 例
- スプリントの振り返りで「レビューが早かった（Keep）」「仕様確認が遅れた（Problem）」「着手前に5分すり合わせる（Try）」とまとめる。
- 個人の週次振り返りで、続けたい習慣・課題・次に試すことを3観点で整理する。
- イベント運営後に、良かった運用・困った点・次回の改善案を棚卸しする。

## 注意点・落とし穴
- Problem の犯人探しにならないようにする。仕組みに目を向ける。
- Try を多く出しすぎると実行されない。次回までに回せる数に絞る。
