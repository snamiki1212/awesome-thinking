# KWLチャート（KWL Chart）

## 一言でいうと
あるテーマについて「**K**：すでに知っていること／**W**：知りたいこと／**L**：学んだこと」の3列を埋めて、学習の前・最中・後を一枚で整理する枠組み。

## 定義
Know（知っていること）/ Want to know（知りたいこと）/ Learned（学んだこと）の3列からなる表（graphic organizer）。学習や調査に入る前に既有知識（K）と問い（W）を書き出し、学習後に得たこと（L）を書き足すことで、自分の知識の変化を可視化する。教育研究者 Donna Ogle が 1986 年に読解の手法として提唱したものが広く知られる（その後 H＝How to learn more を足した KWLH などの拡張もある。整理の一つとして見るとよい）。

## 使いどころ
- 本・記事・資料を読む前後で、理解した内容を整理したいとき。
- 新しいテーマの調査・リサーチで、何を知っていて何を調べるべきかを切り分けたいとき。
- 学習や研修の前後で、自分の理解がどう変わったかを振り返りたいとき。

## 使い方・手順
1. テーマを決め、3列（K / W / L）の表を用意する。
2. **K（Know）**: そのテーマについて今すでに知っていること・思い込みを書き出す。
3. **W（Want to know）**: 知りたいこと・疑問を問いの形で書き出す（K を眺めると問いが出やすい）。
4. ここで学習・調査・読書に取り組む。
5. **L（Learned）**: 学んで分かったことを書く。W の問いに答えが出たか、K の思い込みが正しかったかも確認する。
6. 残った疑問や新たに生まれた問いは、次の W として書き足す。

## 例
- 「発酵食品」を調べる前に、K＝*菌が関わる・味噌や納豆がある*、W＝*なぜ腐敗とは違う? 体にどう良い?* と書く。読んだ後に L＝*微生物が有機物を分解し有用な物質を作るのが発酵* と埋める。
- 研修前に受講者へ K と W を書かせ、研修後に L を書かせて理解の変化と未解決の疑問を可視化する。

## 注意点・落とし穴
- K を埋める段階で「正しく知っていること」だけを書こうとすると手が止まる。思い込み・うろ覚えも書き、後で L と照合して修正することにこそ価値がある。
- W が「はい/いいえ」で終わる浅い問いばかりだと L が痩せる。「なぜ」「どのように」を含む問いを混ぜる。
- 表を埋めること自体が目的化しないよう、L では「W にどう答えたか」「K がどう変わったか」のつながりを意識する。

## 似ている用語との違い
- **KPT（Keep / Problem / Try）** — 3列の表で振り返る点が似るが、KPT は**行動・改善**を回すための振り返り、KWL は**知識・理解**の変化を整理するための学習ツール。
- **5W1H** — 問いを立てて漏れを防ぐ点が似るが、5W1H は1つの事柄を多面的に**記述・把握**する枠、KWL は学習の**前後の変化**を時系列で捉える枠。
- **PDCAサイクル** — 「やる前／やった後」を扱う点が似るが、PDCA は計画→実行の**改善ループ**、KWL は学習を挟んだ**知識の棚卸し**であり繰り返しを主目的としない。
